# Architecture Decision Records

Each non-trivial architectural choice gets a short ADR. An ADR is not bureaucracy — it is a record that future contributors (human or AI) can read to understand _why_ a decision was made.

Format:

```md
# ADR NNNN — <title>

- **Status:** Proposed | Accepted | Superseded by NNNN | Deprecated
- **Date:** YYYY-MM-DD

## Context

## Decision

## Rationale

## Tradeoffs

## Alternatives considered
```

Number ADRs sequentially. If a later ADR supersedes an earlier one, update the earlier one's status and add a forward link.
